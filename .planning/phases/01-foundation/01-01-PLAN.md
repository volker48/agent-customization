    Run `pnpm add minisearch` in the project root.

    **Step 2: Create `pi-extensions/pi-rlm/types.ts`**
    Define shared types used across the extension:

    - `ChunkRecord`: `{ id: string; docId: string; docName: string; text: string; charStart: number; charEnd: number; index: number }` — represents a single chunk with its position in the original document.
    - `DocumentRecord`: `{ id: string; name: string; chunkIds: string[]; totalChars: number; type: string }` — metadata per loaded document.
    - `StoreSnapshot`: `{ chunks: Record<string, ChunkRecord>; documents: Record<string, DocumentRecord>; searchIndexJson: string; buffers: Record<string, unknown> }` — serializable snapshot for `result.details`.
    - `RlmResult<T>`: `{ ok: boolean; data: T; meta: { truncated: boolean; [key: string]: unknown } }` — consistent response envelope per user decision.
    - `DocType`: string union `"python" | "typescript" | "javascript" | "markdown" | "text"` — supported document types.

    **Step 3: Create `pi-extensions/pi-rlm/chunker.ts`**
    Export `chunkDocument(content: string, options: { type?: DocType; chunkSize?: number; overlap?: number }): Array<{ text: string; charStart: number; charEnd: number }>`.

    Implementation:
    - Default chunk size: 1500 characters. Default overlap: 200 characters. Both configurable per user decision (Claude's Discretion).
    - **Type detection** (when `type` not provided): Check for language-specific patterns — `def ` / `class ` for Python, `function ` / `export ` / `class ` for TypeScript/JS, `# ` / `## ` for Markdown. Fall back to `"text"`.
    - **Code splitting** (python, typescript, javascript): Use regex to find top-level function/class definitions. Split at these boundaries. If a single function exceeds chunk size, split at line boundaries within it. Merge small fragments until they approach chunk size. Apply overlap by including trailing characters from previous chunk at start of next.
    - **Prose splitting** (markdown, text): Split on paragraph boundaries (`\n\n`). Merge paragraphs until approaching chunk size. Apply overlap via trailing content from previous chunk.
    - **Oversized handling** (Claude's Discretion): If a single block exceeds chunk size, split at line boundaries. Never split mid-line.
    - Each returned chunk includes `charStart` and `charEnd` as character offsets in the original document.

    Code regex patterns:
    - Python: `/^(?:def |class |async def )/m`
    - TypeScript/JavaScript: `/^(?:export (?:default )?)?(?:function |class |const \w+ = (?:async )?(?:\(|function))/m`
    - Markdown: `/^#{1,6} /m`

    **Step 1: Create `pi-extensions/pi-rlm/store.ts`**
    Export a `DocumentStore` class with these methods:

    **Constructor:** Initialize empty chunk map (`Map<string, ChunkRecord>`), document map (`Map<string, DocumentRecord>`), MiniSearch index, and buffer map (`Map<string, unknown>`). Use a shared `MINISEARCH_CONFIG` constant: `{ fields: ["text"], storeFields: ["chunkId", "docId", "docName", "charStart", "charEnd"] }`. Using `id` as the MiniSearch document ID field (default).

    **`addDocument(content: string, options: { name?: string; type?: DocType; chunkSize?: number; overlap?: number }): DocumentRecord`**
    - Generate docId: SHA-256 hash of content, truncated to 16 hex chars (same as chunk IDs). Per user decision, optional name param; auto-generate hash-based ID if not provided.
    - **Dedup check:** If docId already exists in document map, return existing DocumentRecord (idempotent, per user decision).
    - Call `chunkDocument(content, { type, chunkSize, overlap })` to split.
    - For each chunk: generate chunkId via `createHash("sha256").update(chunk.text).digest("hex").slice(0, 16)`. Store in chunk map. Add to MiniSearch with `{ id: chunkId, text: chunk.text, chunkId, docId, docName, charStart: chunk.charStart, charEnd: chunk.charEnd }`.
    - Create DocumentRecord with chunkIds array, totalChars, type.
    - Return DocumentRecord.

    **`addDocumentBatch(items: Array<{ content: string; options: ... }>): DocumentRecord[]`**
    - Per user decision: all-or-nothing. Validate and chunk all items first. If any fails, throw without mutating state. Then apply all atomically.

    **`search(query: string, options?: { limit?: number }): Array<{ chunkId: string; docId: string; docName: string; score: number; text: string; charStart: number; charEnd: number }>`**
    - Default limit: 10. Configurable per user decision.
    - Call `miniSearch.search(query)` with the limit. Map results back to chunk records.
    - Return results with BM25 scores exposed per user decision.
    - Empty results: return empty array (per user decision, no suggestions).

    **`extractByChunkId(chunkId: string): ChunkRecord | null`**
    - Look up in chunk map. Return full ChunkRecord or null.

    **`extractByRange(docId: string, charStart: number, charEnd: number): string | null`**
    - Find all chunks for docId. Find chunks overlapping the requested range. Reconstruct the span from original content by concatenating overlapping chunk texts and slicing to exact range. Return null if docId not found.
    - Per research recommendation (Claude's Discretion): character ranges are relative to the original document. Walk chunks to find overlapping ones.

    **`saveBuffer(name: string, value: unknown): void`**
    - Store in buffer map. Per Claude's Discretion: accept JSON-serializable values.

    **`getBuffer(name: string): unknown | undefined`**
    - Retrieve from buffer map.

    **`snapshot(): StoreSnapshot`**
    - Serialize: chunks as plain object, documents as plain object, `JSON.stringify(miniSearch)` for search index, buffers as plain object.

    **`restore(snapshot: StoreSnapshot): void`**
    - Clear all maps. Rebuild chunk map and document map from snapshot. Restore MiniSearch via `MiniSearch.loadJSON(snapshot.searchIndexJson, MINISEARCH_CONFIG)`. Restore buffers.

    **`reset(): void`**
    - Clear all maps and recreate empty MiniSearch index.
